Registration:
    User submits form → Validate fields → Hash password → Save to Users DB
    → If role is citizen: account_status = 'pending' (auto or active)
    → If role is vet/shelter/civic: account_status = 'pending' → Needs SuperAdmin approval

    User submits email + password → Validate credentials against DB
    → If valid and account_status == 'active': Generate JWT (HS256, 30min expiry)
    → Return {access_token, refresh_token, user role}
    → Frontend stores token in AuthContext → Redirects to role-based dashboard

PART B: ALGORITHMS
Algorithm 1: Role-Based Access Control (RBAC)
Module: All API Endpoints (Middleware) Purpose: Prevent unauthorized access to role-restricted endpoints.
Pseudocode:
ALGORITHM: RBAC_Verify
INPUT: request (Authorization header, endpoint)
OUTPUT: ALLOW or DENY
BEGIN
    token = request.headers["Authorization"].split("Bearer ")[1]
      TRY
        decoded = JWT.decode(token, SECRET_KEY, "HS256")
    CATCH ExpiredError, DecodeError
        RETURN DENY (401: "Invalid or expired token")
    END TRY
    user = DB.get(User, id=decoded.user_id)
    role = user.role
    PERMISSION_MATRIX = {
        "citizen":        [Pet_CRUD, SOS_Create, Adoption_Apply],
        "veterinarian":   [Clinical_CRUD, Prescription_Create, Media_Upload],
        "shelter_admin":  [Inventory_CRUD, Application_Review],
        "civic_authority": [Analytics_View, Rescue_Dispatch],
        "admin":          [ALL_ACCESS]   }
    IF request.endpoint IN PERMISSION_MATRIX[role] THEN
        RETURN ALLOW
    ELSE
        RETURN DENY (403: "Forbidden")
    END IF
END
Complexity: Time O(n) where n = permissions per role, Space O(1)
Algorithm 2: Pet Duplicate Detection
Module: Citizens (apps.citizens) Purpose: Prevent accidental re-registration of the same pet by the same owner.
Pseudocode:
ALGORITHM: Register_Pet
INPUT: owner_id, pet_data (name, species, breed, dob, photo)
OUTPUT: new_pet_record OR duplicate_warning
BEGIN
    IF pet_data.name IS EMPTY OR pet_data.species IS EMPTY THEN
        RETURN Error(400, "Name and Species required")
    END IF
    existing = DB.query(
        "SELECT * FROM Pet WHERE owner_id = ? 
         AND LOWER(name) = LOWER(?) AND species = ?",
        owner_id, pet_data.name, pet_data.species
    )
    IF existing IS NOT EMPTY THEN
        RETURN Warning("Similar pet already registered")
    END IF
    IF pet_data.photo EXISTS THEN
        image_url = S3.upload(pet_data.photo)
    END IF
    new_pet = DB.insert("Pet", {
        owner_id, name, species, breed, dob, media_url: image_url
    })
    RETURN new_pet
END
Complexity: Time O(n) where n = owner's pets, Space O(1)
Algorithm 3: Predictive Health Risk Flagging
Module: Analytics (apps.analytics) Purpose: Detect overweight/underweight risks by comparing a pet's current weight against breed-specific baselines.
Pseudocode:
ALGORITHM: Health_Risk_Analysis
INPUT: all_pets (list of Pet records)
OUTPUT: flags_created (integer count)
BEGIN
    flags_created = 0
    BREED_BASELINES = {
        "Golden Retriever": {ideal: 30.0 kg, variance: 15%},
        "Labrador":         {ideal: 32.0 kg, variance: 15%},
        "Pug":              {ideal: 8.0 kg,  variance: 10%}
    }
    FOR EACH pet IN all_pets DO
        latest_record = pet.medical_records.latest("date")
        current_weight = latest_record.weight_kg OR pet.current_weight_kg
        baseline = BREED_BASELINES.get(pet.breed)
        IF baseline IS NULL THEN CONTINUE
        upper_limit = baseline.ideal * (1 + baseline.variance)
        lower_limit = baseline.ideal * (1 - baseline.variance)
        IF current_weight > upper_limit THEN
            severity = "High" IF current_weight > upper_limit * 1.1 ELSE "Medium"
            DB.insert("HealthRiskFlag", {
                pet, risk_type: "Overweight", severity,
                description: "Weight exceeds safe range for breed"
            })
            flags_created += 1
        ELSE IF current_weight < lower_limit THEN
            DB.insert("HealthRiskFlag", {
                pet, risk_type: "Underweight", severity: "Medium",
                description: "Weight below safe threshold for breed"
            })
            flags_created += 1
        END IF
    END FOR
    RETURN flags_created
END
Complexity: Time O(n) where n = total pets, Space O(1)
Algorithm 4: SHA-256 Record Immutability (Vaccination & Prescription)
Module: Clinical (apps.clinical) Purpose: Ensure clinical vaccination and prescription records cannot be tampered with after finalization by generating a cryptographic hash.
Pseudocode:
ALGORITHM: Freeze_Clinical_Record
INPUT: record (VaccinationLog or DigitalPrescription)
OUTPUT: record with record_hash set
BEGIN
    IF record.is_frozen == TRUE AND record.id IS NULL (new record) THEN
        // Build a deterministic data string
        IF record IS VaccinationLog THEN
            data_string = record.vaccine_name + "-" + 
                          record.batch_number + "-" + 
                          record.date_administered
        ELSE IF record IS DigitalPrescription THEN
            data_string = STRING(record.medications) + 
                          STRING(record.issued_date)
        END IF
        // Generate SHA-256 hash
        record.record_hash = SHA256(data_string.encode("utf-8")).hexdigest()
    END IF
    DB.save(record)
    RETURN record
END
VERIFICATION ALGORITHM:
    To verify integrity later:
    recompute_hash = SHA256(same_data_string)
    IF recompute_hash == record.record_hash THEN
        RETURN "Record is AUTHENTIC"
    ELSE
        RETURN "Record has been TAMPERED"
    END IF
Complexity: Time O(1), Space O(1)

Algorithm 5: Adoption Application State Machine
Module: Shelter (apps.shelter) Purpose: Manage the lifecycle of an adoption application through defined states.
Pseudocode:
ALGORITHM: Process_Adoption_Application
INPUT: application_id, new_status, admin_user
OUTPUT: updated_application
BEGIN
    app = DB.get("AdoptionApplication", id=application_id)
    // Define valid state transitions
    VALID_TRANSITIONS = {
        "Pending"     → ["Under Review", "Cancelled"],
        "Under Review" → ["Interview Scheduled", "Approved", "Rejected"],
        "Interview Scheduled" → ["Approved", "Rejected"],
        "Approved"    → [],    // Terminal state
        "Rejected"    → [],    // Terminal state
        "Cancelled"   → []     // Terminal state
    }
    IF new_status NOT IN VALID_TRANSITIONS[app.status] THEN
        RETURN Error(400, "Invalid state transition")
    END IF
    app.status = new_status
    DB.save(app)
    // If approved, update animal status
    IF new_status == "Approved" THEN
        animal = app.animal
        animal.is_adopted = TRUE
        animal.is_available = FALSE
        DB.save(animal)
    END IF
    // Log to audit trail
    Log_Audit_Trail(admin_user, "APPLICATION_" + new_status, app)
    RETURN app
END
Complexity: Time O(1), Space O(1)
